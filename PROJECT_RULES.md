# Project Rules

- Framework stays `Next.js` unless explicitly changed.
- Check official documentation before larger framework-level changes.
- Prefer stable App Router patterns over clever shortcuts.
- Do not break working sections without a clear reason.
- Favor maintainable, production-minded code over quick hacks.
- Treat the owner as non-technical: explain decisions clearly and keep the implementation path simple.
- Use English as the primary site language for US outreach.
- Position the studio around websites and landing pages first; AI stays an add-on unless explicitly promoted.
